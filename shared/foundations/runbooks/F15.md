# F15 — Inter-agent misalignment

Failure code from `taxonomy/f15-inter-agent-misalignment.md`.

## Signature
Two or more subagents pursue contradictory subgoals; each reports success; the orchestrator composes their outputs without detecting the conflict.

## Detection
- **Audit-trail signal:** composed artifact contains contradictory clauses, conflicting config keys, or overlapping metric targets from different subagent returns.
- **Metrics signal:** downstream error rate spikes immediately after a parallel subagent batch; structured return fields from two subagents have overlapping keys with incompatible values.
- **Self-report signal:** orchestrator declares task done without a reconciliation turn; no `conflicts_with` field is present in any subagent return.

## Triage
1. Identify the two (or more) subagents whose returns are logically incompatible.
2. State the contradiction in one sentence: "Subagent A required X; Subagent B required not-X."
3. Determine whether the conflict is in goals (assign different objectives were never cross-checked) or in shared state (both wrote to the same artifact).

## Rollback
- **If during a skill run:** halt composition; do not merge subagent outputs; surface the conflict to the orchestrator for explicit resolution.
- **If during a deployed plugin:** revert the composed artifact to its pre-batch state; rerun each subagent with an explicit `conflicts_with` requirement in its structured return clause.
- **State to verify before resuming:** each subagent's return includes a `conflicts_with` field; the orchestrator has checked all non-empty `conflicts_with` values and resolved them before recomposing.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances in one workflow, the orchestrator's composition step has no reconciliation layer — redesign before continuing; escalate to the project's failure-log review.
