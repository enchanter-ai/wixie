# F12 — Degeneration loop

Failure code from `../conduct/failure-modes.md`.

## Signature
Same edit, reverted, re-applied across iterations.

## Detection
- **Audit-trail signal:** an iteration's diff equals (or near-equals) a diff reverted earlier in the same prompt's lifecycle; convergence variance σ explodes after a tightening pass.
- **Metrics signal:** axis-N score oscillates +/- around a fixed mean across 3+ iterations; no-regression contract triggers more than once on the same axis.
- **Self-report signal:** "let me try the X tightening again", repeated across rounds without addressing why it failed last time.

## Triage
1. Pull `learnings.md` for the affected prompt; find prior F12 entries on the same axis.
2. Confirm the current hypothesis is materially different from the reverted ones.
3. If not, switch axis per the no-regression contract.

## Rollback
- **If during a skill run:** auto-revert the iteration; mark the dimension saturated; pick a different axis next round.
- **If during a deployed plugin:** freeze convergence on this prompt; record the dead-end in `learnings.md`; do not retry the same axis until a new mechanism is identified.
- **State to verify before resuming:** the next attempt targets a different axis or applies a meaningfully different hypothesis.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, freeze convergence on this prompt per `failure-modes.md` § Escalation patterns. Escalate to the project's failure-log review.
