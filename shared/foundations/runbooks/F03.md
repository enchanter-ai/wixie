# F03 — Context decay

Failure code from `../conduct/failure-modes.md`.

## Signature
Instruction from top of context violated at bottom.

## Detection
- **Audit-trail signal:** assistant turn produces output that contradicts a constraint set in the first ~200 tokens of the same session.
- **Metrics signal:** assertion-pass rate degrades as session length grows past ~50% of the model's effective context window.
- **Self-report signal:** "I forgot the earlier rule", or output format shifts mid-session without an explicit instruction change.

## Triage
1. Identify the violated instruction and where it lives in context (top, middle, bottom).
2. Confirm the violation is decay (instruction still present) vs. an explicit override turn.
3. Emit a checkpoint per `../conduct/context.md` § Checkpoint protocol.

## Rollback
- **If during a skill run:** drop prior turns, re-anchor to the checkpoint, and re-attempt the failing step.
- **If during a deployed plugin:** restart the skill with a tighter initial context (smallest-set rule); move the violated rule to top-200 or bottom-200 slot.
- **State to verify before resuming:** the constraint appears in the U-curve recall slots, not the middle.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, escalate to the project's failure-log review.
