# F17 — System-design brittleness

Failure code from `taxonomy/f17-system-design-brittleness.md`.

## Signature
An agent network fails on a single-node drop because no graceful degradation path was designed in; the orchestrator hangs or surfaces an opaque error with no partial-return.

## Detection
- **Audit-trail signal:** orchestrator session ends without a final structured return; intermediate subagent results are lost; the error message names a tool timeout or missing dependency but no partial-success path was taken.
- **Metrics signal:** pipeline failure rate tracks with availability of one specific node or API; removing that node produces a full failure rather than a degraded output.
- **Self-report signal:** no `timeout_behavior` clause appears in any subagent prompt; no fallback agent role is named in the pipeline design.

## Triage
1. Identify the dropped node: which subagent, tool, or API was unavailable.
2. Confirm whether the node had a `timeout_behavior` clause; if not, that is the design gap.
3. Assess what completed work was lost and whether it is recoverable from partial returns.

## Rollback
- **If during a skill run:** collect all subagent returns already received; surface them as a partial result with an explicit gap flag naming the dropped node.
- **If during a deployed plugin:** add a `timeout_behavior` clause and a named fallback to the dropped node's delegation prompt before rerunning; do not rerun without a degradation path.
- **State to verify before resuming:** each subagent prompt includes a `timeout_behavior` clause; each agent role has a named fallback or a partial-return path; the orchestrator is tested against a simulated node drop before the next production run.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md` with the dropped node identified.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances in one workflow, the pipeline has no graceful degradation layer — redesign with explicit timeout behavior and partial-return paths before continuing; escalate to the project's failure-log review.
