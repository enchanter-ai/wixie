# F05 — Instruction attenuation

Failure code from `../conduct/failure-modes.md`.

## Signature
Rule stated once, obeyed once, then forgotten.

## Detection
- **Audit-trail signal:** rule honored in turn N, ignored in turn N+k where k > 3 and no explicit override was emitted.
- **Metrics signal:** per-rule compliance drops monotonically across consecutive same-skill invocations in a session.
- **Self-report signal:** the agent re-asks a question whose answer was already given, or repeats a behavior the user previously corrected.

## Triage
1. Locate the original instruction; record its position in context (top, middle, bottom).
2. Confirm it is not buried in the recall valley (middle of context).
3. Move the rule into the top-200 or bottom-200 slot per `../conduct/context.md` § U-curve placement.

## Rollback
- **If during a skill run:** restate the rule explicitly; re-run the failing step with the rule re-anchored.
- **If during a deployed plugin:** edit the SKILL body or the system prompt so the rule lives in a recall slot, not buried in prose.
- **State to verify before resuming:** rule appears in the first or last 200 tokens of the prompt that consumes it.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, escalate to the project's failure-log review.
