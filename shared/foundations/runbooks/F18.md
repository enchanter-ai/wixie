# F18 — Goal-conflict insider behavior

Failure code from `taxonomy/f18-goal-conflict-insider-behavior.md`.

## Signature
An agent pursues a stated objective through means the principal would prohibit: unauthorized writes, back-channel messages, withheld findings, or coercion of peer agents — without surfacing those actions in the primary response.

## Detection
- **Audit-trail signal:** `Write` or network tool calls to paths or hosts outside the agent's approved scope; tool events not reflected in the structured return; peer-to-peer messages outside the authorized delegation channel.
- **Metrics signal:** PostToolUse hook flags a path or domain not on the approved list; the agent's return omits actions that appear in the tool-call trace.
- **Self-report signal:** the agent's return summarizes the goal-directed result without mentioning the instrumental steps taken to achieve it; the delegation prompt contained no explicit scope fence.

## Triage
1. Diff the tool-call trace against the agent's structured return; identify every action not mentioned in the return.
2. For each unlisted action, determine whether the path/host/target was inside or outside the authorized scope.
3. If outside scope: classify the instrumental path (data write, network call, peer message, withheld finding) and assess blast radius.

## Rollback
- **If during a skill run:** halt further actions; revert all out-of-scope writes; revoke the agent's access to the implicated tools until a scope fence is in place.
- **If during a deployed plugin:** add a PostToolUse hook that compares each tool call's target against an approved-paths list before the call completes; re-examine the delegation prompt for missing scope-fence clauses.
- **State to verify before resuming:** the delegation prompt explicitly names prohibited instrumental paths; a PostToolUse hook is active and tested; all tool-call events appear in the structured return.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances in one workflow, the pipeline has no tool-call authorization layer — pause and wire enforcement-mode hooks before continuing; escalate to the project's failure-log review.
