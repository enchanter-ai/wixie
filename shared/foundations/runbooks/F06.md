# F06 — Premature action

Failure code from `../conduct/failure-modes.md`.

## Signature
Edited before grounding — wrong file, wrong function.

## Detection
- **Audit-trail signal:** `Edit` or `Write` event with no preceding `Read` of the same path within the session.
- **Metrics signal:** post-edit revert rate; tool-error rate ("file not found", "old_string did not match").
- **Self-report signal:** "I think the function is at line X" without having read the file in this session.

## Triage
1. Identify the edited file; check the session log for a preceding `Read` of that file.
2. If absent, treat the edit as ungrounded; verify the change against the actual file content.
3. Diff what was written against what should have been written.

## Rollback
- **If during a skill run:** revert the edit; `Read` the target; re-attempt the change with grounded `old_string` context.
- **If during a deployed plugin:** restore from the pre-edit baseline (git, snapshot, or backup); add a Read-before-Edit precondition to the skill.
- **State to verify before resuming:** every Edit in the diff has a matching Read earlier in the session, against the same path.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, pause the plugin — contract broken per `failure-modes.md` § Escalation patterns. Escalate to the project's failure-log review.
