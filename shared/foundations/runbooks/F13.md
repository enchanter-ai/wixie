# F13 — Distractor pollution

Failure code from `../conduct/failure-modes.md`.

## Signature
Long irrelevant context bent the output.

## Detection
- **Audit-trail signal:** large `Read` / `WebFetch` payloads earlier in the session followed by output that adopts vocabulary, structure, or claims from those payloads despite being off-topic.
- **Metrics signal:** topic-drift score elevated; off-topic noun/term frequency in output > baseline; assertion-pass rate drops as session token count grows.
- **Self-report signal:** the output references material the user did not ask about; the agent paraphrases an earlier-fetched page without being prompted.

## Triage
1. Identify the polluting payload(s) — large attachments, full file dumps, broad WebFetch results.
2. Confirm the payload is not load-bearing for the current task.
3. Apply the smallest-set rule per `../conduct/context.md` § Smallest-set rule.

## Rollback
- **If during a skill run:** drop the polluting context via a checkpoint; restart the failing step with only the necessary excerpts.
- **If during a deployed plugin:** narrow `Read` calls with `offset`/`limit`; replace full attachments with 400-token excerpts; route bulky reads to a subagent that returns a structured summary.
- **State to verify before resuming:** context is dominated by signal — the top, middle, and bottom of the window each describable in one line.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, escalate to the project's failure-log review.
