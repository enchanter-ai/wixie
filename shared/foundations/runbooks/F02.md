# F02 — Fabrication

Failure code from `../conduct/failure-modes.md`.

## Signature
Cited API / flag / file that doesn't exist.

## Detection
- **Audit-trail signal:** `Edit`/`Bash` calls referencing paths that fail with `not found`, or `WebFetch` quotes that don't match the source page on re-fetch.
- **Metrics signal:** post-action error rate spike; cite-verify check fails (quote not present in cited URL).
- **Self-report signal:** confident assertion of a flag/path/function name without preceding `Read`, `Grep`, or `Glob`.

## Triage
1. Identify the specific cited entity (path, flag, function, URL, quote).
2. Run `Glob` / `Grep` / `WebFetch` to verify existence.
3. If missing: stop downstream consumption immediately — fabricated context cascades.

## Rollback
- **If during a skill run:** revert any artifact that consumed the fabricated cite; re-run from the last verified state.
- **If during a deployed plugin:** mark the affected output `unverified`; re-execute with a verification step prepended.
- **State to verify before resuming:** every external reference in the in-flight artifact has a passing existence check.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, escalate to the project's failure-log review.
