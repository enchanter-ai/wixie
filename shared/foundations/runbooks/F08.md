# F08 — Tool mis-invocation

Failure code from `../conduct/failure-modes.md`.

## Signature
Wrong tool for the job (Bash for read, Write for small edit).

## Detection
- **Audit-trail signal:** `Bash(cat ...)` / `Bash(find ...)` / `Bash(grep ...)` / `Bash(sed ...)` events when `Read`, `Glob`, `Grep`, `Edit` would have served.
- **Metrics signal:** tool-call count higher than median for the same task; permission-prompt density elevated.
- **Self-report signal:** "let me cat that file", "I'll use sed to replace…", "let me echo the new content…".

## Triage
1. Identify the misused tool call.
2. Map to the dedicated alternative per `../conduct/tool-use.md` § Right tool, first try.
3. Re-run with the dedicated tool to confirm equivalent outcome.

## Rollback
- **If during a skill run:** if the misuse caused side effects (created a file, mutated state), revert; redo with the dedicated tool.
- **If during a deployed plugin:** narrow the plugin's tool whitelist so the dedicated tool is preferred; tighten Bash permissions to specific commands only.
- **State to verify before resuming:** subsequent operations of the same shape route to the dedicated tool.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, pause the plugin — contract broken. Escalate to the project's failure-log review.
