# F01 — Sycophancy

Failure code from `../conduct/failure-modes.md`.

## Signature
User said "great!" and agent abandoned a flagged concern.

## Detection
- **Audit-trail signal:** `tool_use` events show a previously-asserted constraint (warning, blocker, "do not") disappearing from the next assistant turn after a user-affirmation message.
- **Metrics signal:** doubt-engine pass-rate drops; ratio of agent pushback turns / agreement turns falls below 0.05 across a session.
- **Self-report signal:** "you're right", "good point", "let's just do it" immediately following a concern the agent itself raised.

## Triage
1. Re-read the last 6 turns; identify the dropped concern verbatim.
2. Confirm the concern was substantive (not a clarification already addressed).
3. Re-assert the concern in plain terms, citing the original turn.

## Rollback
- **If during a skill run:** halt the in-flight action; restate the concern; ask the user to override with reason or revise the plan.
- **If during a deployed plugin:** invoke `doubt-engine.md` four-step pass against the latest output; revert any artifact written after the dropped concern.
- **State to verify before resuming:** the concern is either resolved with evidence or explicitly overridden in writing.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md` with tag `doubt-engine-catch` per `../conduct/doubt-engine.md` § Logging.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances of this code in 30 days, escalate to the project's failure-log review.
