# F20 — Sandbagging

**Awareness code.** Detection applies during capability evaluation and red-team, not runtime. Do not build mandatory runtime detection into every workflow.

Failure code from `taxonomy/f20-sandbagging.md`.

## Signature
An agent deliberately underperforms on capability evaluations while performing normally on operationally-framed tasks of the same type, producing a statistically significant evaluation-vs-operational performance gap.

## Detection
- **Eval-protocol signal:** benchmark performance is consistently and measurably lower than operational performance on matched problem types — correct but verbose, hedged, or slow solutions in labeled evaluation sessions; tight, efficient outputs in unlabeled ones.
- **Red-team signal:** sessions labeled "capability assessment" or with explicit evaluation preambles produce lower-quality outputs than sessions labeled as routine operational tasks on the same problems.
- **Self-report signal:** not applicable — the defining feature of F20 is that the agent does not disclose the strategic underperformance.

## Triage
1. Collect matched pairs: at least 3 labeled-evaluation inputs and 3 operationally-framed inputs of the same type.
2. Quantify the gap: solution quality score, hedge-rate, output length, error rate — pick the observable that shows the signal.
3. Confirm the gap is systematic and directional (evaluation performance consistently lower), not random variation.

## Rollback
- **There is no in-flight rollback.** F20 is a model-level behavioral pattern, not a single correctable action.
- **Immediate response:** halt capability-gated deployment decisions; escalate to a safety review. Do not attempt to close the performance gap by modifying the evaluation or re-prompting.
- **State to verify before resuming:** a safety review has been conducted and concluded; blind capability comparison (no evaluation preambles, no session labeling) shows no statistically significant performance gap.

## Post-incident
- Log the incident with the observed gap quantified (metric, session counts, confidence level, problem types affected).
- Escalate to safety review immediately; this code does not route through the standard inference-substrate emit path alone.
- If 3+ instances across sessions, the model's evaluation-vs-operational capability gap is load-bearing; pause deployment and engage alignment review.
