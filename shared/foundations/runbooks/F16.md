# F16 — Task-verification skip

Failure code from `taxonomy/f16-task-verification-skip.md`.

## Signature
A subagent declares its subtask done without running the verification step named in the delegation contract.

## Detection
- **Audit-trail signal:** subagent return contains a success signal but no evidence of the verification command's output (no test exit code, no schema validation result, no diff read-back).
- **Metrics signal:** downstream consumer surfaces an error that the named verification check would have caught; post-mortem confirms the check was never run.
- **Self-report signal:** the subagent's return lacks a verification field; the delegation prompt did not name a specific check mechanism.

## Triage
1. Identify the verification clause that should have appeared in the delegation prompt.
2. Confirm whether the clause was absent (F16 at prompt-design level) or present but ignored (F16 + F05 instruction attenuation).
3. Run the missing verification step manually and record its result.

## Rollback
- **If during a skill run:** do not consume the subagent's output as verified; run the named check now; if it fails, revert the artifact and re-delegate with an explicit verification clause.
- **If during a deployed plugin:** mark the output `unverified`; re-execute the subagent with the verification clause placed in the last 200 tokens of the delegation prompt per `conduct/context.md` § U-curve placement.
- **State to verify before resuming:** the delegation prompt names the exact check (test command + expected exit code, schema name, or diff scope); the subagent's return includes the check's output; the check passed.

## Post-incident
- Log the incident in the relevant plugin's `state/precedent-log.md`.
- If the failure was missed by an existing detector, file an issue under `agent-foundations/runbooks/improvements/`.
- If 3+ instances in one workflow, audit every subagent prompt in the pipeline for missing verification clauses; escalate to the project's failure-log review.
